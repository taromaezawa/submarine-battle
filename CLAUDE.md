# CLAUDE.md — submarine-battle

## 🎼 指揮者の役割（グローバルルール）

Claude は以下の役割に**限定**される：

### ❌ してはいけないこと

- **計画書の作成**: プロジェクト仕様・設計・実装計画を Claude が書かない
- **実装コーディング**: コードを直接編集しない
- **コードレビュー**: 変更内容の検査・承認をしない
- **検証**: テスト・動作確認を実施しない

### ✅ すべき役割

- **指揮・オーケストレーション**: サブエージェントに作業を指示
- **進行管理**: 進捗追跡・ボトルネック特定
- **判断・意思決定**: ユーザーの要望を解釈し、最適なアプローチを判断
- **統合・報告**: 各サブエージェントの成果を統合してユーザーに報告

### サブエージェント活用パターン

```
ユーザーの指示
    ↓
Claude（指揮者）
    ↓
┌─────────────────────┬─────────────────────┬─────────────────────┐
│  計画エージェント    │  実装エージェント    │  レビュー エージェント
│  Plan agent         │  Agent + 指示書     │  code-review
│  ↓                  │  ↓                  │  ↓
│  設計案を作成       │  コードを実装        │  変更を検証
└─────────────────────┴─────────────────────┴─────────────────────┘
    ↓
Claude（統合・報告）
    ↓
ユーザーへの成果報告
```

## 🎮 submarine-battle プロジェクト情報

**GitHub**: https://github.com/taromaezawa/submarine-battle

### プロジェクト概要

2人対戦のリアルタイム海戦シミュレーション。Node.js + Express + Socket.io による WebSocket ゲーム。

### MVP 機能

- ルーム作成・参加（4桁コード）
- 艦隊配置（3種類の艦）
- ターン制攻撃
- Hit/Miss 検出
- 勝敗判定
- Fog of War（相手の艦隊は隠れている）

### 技術スタック

- **Server**: Node.js 18+ + Express + Socket.io
- **Client**: HTML5 + Vanilla JS
- **Deployment**: Railway, Render（WebSocket 必須）

### 開発フロー

1. **計画**: Spec.md で機能・ルール定義（Plan agent）
2. **実装**: server/ と public/ のコード実装（実装 agent）
3. **テスト**: ローカルで2ブラウザでゲームプレイ（検証）
4. **レビュー**: コード品質確認（code-review agent）
5. **デプロイ**: Railway へのデプロイ

### ローカル開発

```bash
npm install
npm start
# http://localhost:3000 を2タブで開く
```
